Hyperledger Fabric 특징

  • 허가형 네트워크 (허가된 노드만 네트워크 참여 가능)
  • 채널 : 네트워크 내 특정 참여자들만을 위한 논리적 분할

보증 정책과 Orderer

  • 신뢰 시스템
  • 체인코드 보증 정책 ex) AND(노드1, 노드2)
  • 실행 결과가 다를 경우

체인코드 작성 ㅇ의 사항

  • 결정론적으로 작성(보증 정책)
  • 외부 값 쿼리, 난수 발생기, 맵 이터레이션 등 다를 수 있음
  • 내부 변수 X
  • 상태 변경이 바로 되지 않음
  • 같은 키에 대해서 두번 업데이트 불가 (토큰 utxo 방식등 차용해야함)
